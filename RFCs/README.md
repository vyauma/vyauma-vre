# Vyauma Request for Comments (RFCs)

The "RFC" (request for comments) process is intended to provide a consistent and controlled path for new features to enter the language and standard library, so that all stakeholders can be confident about the direction the language is evolving in.

## When to follow this process

You need to follow this process if you intend to make "substantial" changes to Vyauma or its documentation. Some examples that would require an RFC are:
- A new feature that creates new syntactic or semantic surface area.
- Removing features that already shipped as part of the release channel.
- The introduction of new core-provided tooling.

## The Process
1. Copy `template.md` to `text/0000-my-feature.md` (where 'my-feature' is descriptive).
2. Fill in the RFC.
3. Submit a pull request.
4. Build consensus and integrate feedback.
